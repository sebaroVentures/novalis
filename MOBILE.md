# Novalis on Mobile (Android / iOS)

Novalis is built with Tauri v2, which targets Android and iOS from the **same**
`apps/desktop/src-tauri` crate and the same React frontend — there is no separate
mobile app. The mobile entry point is already wired (`run()` is annotated with
`#[cfg_attr(mobile, tauri::mobile_entry_point)]`).

> **Status.** The codebase is mobile-ready and the desktop build stays green, but
> the Android/iOS targets have **not** been built or tested here — that requires
> platform SDKs and devices. This doc is the runbook to take it the rest of the
> way on a suitably set-up machine.

## What's already done for mobile

- **One codebase, both targets** — same Rust core, same UI.
- **Desktop-only bits are gated.** The `notify` file watcher is behind
  `#[cfg(desktop)]` (it isn't built for mobile). On mobile, refresh happens via
  the **Refresh (⟳)** button → `rescan_vault` (call it on app resume too).
- **Responsive UI** — the sidebar collapses into a drawer on narrow screens with
  a top-bar menu button.
- **Icons** — `src-tauri/icons/` already includes the Android `mipmap-*` and iOS
  `AppIcon-*` assets (generated by `tauri icon`).

## Prerequisites

**Android:** Android Studio (SDK + NDK), JDK 17, and env vars `ANDROID_HOME` and
`NDK_HOME`. Add the Rust targets:
`rustup target add aarch64-linux-android armv7-linux-androideabi i686-linux-android x86_64-linux-android`.

**iOS (macOS only):** Xcode + command-line tools, CocoaPods, and Rust targets
`rustup target add aarch64-apple-ios aarch64-apple-ios-sim x86_64-apple-ios`.
A signing team/provisioning profile is needed for physical devices.

## Bootstrap & run

From `apps/desktop/`:

```bash
pnpm android:init      # generates src-tauri/gen/android  (run once)
pnpm android:dev       # build + run on an emulator/device

pnpm ios:init          # generates src-tauri/gen/apple    (run once, macOS)
pnpm ios:dev
```

`gen/` is git-ignored; commit it only if you want to hand-edit native projects.

## Remaining work (device-dependent)

1. **Vault file access.** The desktop folder picker and absolute vault paths don't
   map cleanly to mobile sandboxes. Plan: on mobile, use the platform document
   picker / app documents directory and persist a security-scoped bookmark
   (iOS) / URI permission (Android) via `tauri-plugin-fs`, then pass the resolved
   path to `open_vault`. `novalis-core` already takes a plain `&Path`, so the core
   is unaffected.
2. **Foreground refresh.** Call `rescan_vault` on app resume (Tauri lifecycle /
   visibilitychange) so external sync changes show without a watcher.
3. **OAuth on mobile.** The loopback redirect should be swapped for a deep-link
   (custom scheme) redirect via `tauri-plugin-deep-link` on mobile.
4. **Signing / release.** Android keystore + iOS provisioning; desktop
   notarization (macOS) and signing (Windows). Wire into `release.yml`.
